## 计划包含的示例场景

**场景一：从零启动一个新产品**
展示完整的 Define → Implement 流程，包括 Mockup 确认迭代、verify 发现矛盾并修复的过程。

**场景二：已有产品的功能迭代**
展示 PRD 迭代模式（在 PRD_V1.md 上追加迭代），以及如何判断哪些下游文档需要跟着更新。

**场景三：verify 发现跨文档矛盾**
展示 Global Verifier 的输出，以及如何定位矛盾来源、修复对应文档、重新通过 verify。

**场景四：execute 过程中发现规格冲突**
展示 Exe Agent 暂停、上报冲突、等待确认的正确处理方式，对比自行假设导致漂移的反面案例。
